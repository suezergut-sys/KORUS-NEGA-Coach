# DUEL — голосовой тренажёр переговоров

Прототип русскоязычного тренажёра управленческих переговоров. Его первая задача — проверить, насколько естественно и быстро работает двусторонний голосовой диалог через OpenAI Realtime API.

## Что уже работает

- браузер передаёт микрофон напрямую в OpenAI по WebRTC;
- `gpt-realtime-2` отвечает голосом и допускает перебивания;
- семантическое определение конца реплики;
- живая русская расшифровка;
- замер времени от конца реплики пользователя до первого аудиофрагмента ответа;
- выбор роли, сложности и контекста кейса;
- приватная база книги в Supabase Storage и 153 фрагмента в pgvector;
- 34 кандидата методических атомов со статусом `candidate`;
- сохранение стенограммы и структурированный пост-анализ после завершения;
- проверка доказательств: цитата оценки должна дословно присутствовать в книге и стенограмме;
- секретный API-ключ остаётся только на сервере.

Оригинал книги Владимира Тарасова хранится локально и в приватном Supabase Storage. Методические атомы пока являются кандидатами и требуют экспертной верификации, поэтому интерфейс маркирует оценку как предварительную. Технические сведения об источнике находятся в `docs/SOURCE_MANIFEST.md`.

## Запуск

1. Скопируйте `.env.example` в `.env.local`.
2. Заполните `OPENAI_API_KEY`, `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY`.
3. Установите зависимости: `npm install`.
4. Примените миграцию из `supabase/migrations/`.
5. Импортируйте книгу: `npm run methodology:ingest`.
6. При необходимости извлеките кандидаты правил: `npm run methodology:extract`.
7. Запустите: `npm run dev`.
8. Откройте `http://localhost:3000` и разрешите доступ к микрофону.

## Деплой на Vercel

Добавьте `OPENAI_API_KEY`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` и `OPENAI_ANALYSIS_MODEL` в Environment Variables проекта Vercel. Для работы микрофона нужен HTTPS; Vercel предоставляет его автоматически.

Полное проектное решение описано в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
